概要

Joom Merchant API v3は、マーチャントや開発者がJoomプラットフォームと連携するためのAPIです。このAPIを使用することで、Web版Joom for Merchantsの機能をプログラムから完全に操作できます。

主な用途：
  • 商品の作成・管理
  • 注文の処理・発送
  • ERPシステムとの連携
  • 在庫管理の自動化

ベースURL

本番環境：
https://api-merchant.joom.com/api/v3

※サンドボックス環境については公式ドキュメントを参照してください。

OAuth 2.0 認証

JoomはOAuth 2.0認証を採用しており、セキュアなAPI連携を実現しています。これはFacebookやWeChatなどでログインする際と同じ認証プロセスです。

主要な用語

用語 説明
App/Client（アプリ/クライアント） Joom上のマーチャントデータにアクセスしたいアプリケーション
User（ユーザー） Joomにマーチャントアカウントを持つ人。アプリにデータアクセスの許可を与える人
Access Token（アクセストークン） Joomがアプリに発行する固有の文字列。ユーザーデータにアクセスする際の識別に使用

認証リクエストの方法

すべてのAPIリクエストにはアクセストークンが必要です。リクエストヘッダーにトークンを追加してください。

// リクエストヘッダーの例 Authorization: Bearer YOUR_ACCESS_TOKEN Content-Type: application/json

リクエストの基本

重要なルール

  • HTTPS必須：すべてのリクエストはHTTPSで行う必要があります。HTTPでのリクエストは拒否されます。
  • Content-Type：POSTエンドポイントでJSONボディを送信する場合、Content-Type: application/jsonヘッダーが必要です。
  • URLパラメータ：一部のエンドポイントはURLクエリパラメータを受け付けます。

リクエスト例

// cURLでのリクエスト例 curl -X GET "https://api-merchant.joom.com/api/v3/products" \ -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \ -H "Content-Type: application/json"

商品（Products）API

商品の作成、更新、取得に関するエンドポイントです。

商品識別子について

識別子 説明 生成元
Product SKU 商品の固有識別子（マーチャント全体で一意） マーチャント
Variant SKU バリエーションの固有識別子（マーチャント全体で一意） マーチャント
Joom Product ID Joomが発行する商品ID Joom
Joom Variant ID Joomが発行するバリエーションID Joom
ポイント： SKUのみを使用してすべての操作が可能ですが、Joom IDを使用することもできます。

商品のバージョン

各商品には2つのバージョンがあります：

  • 現行バージョン（Current）：最新の変更を反映（下書き状態）
  • 承認済みバージョン（Approved）：公開されているバージョン

主要エンドポイント

POST /products

新しい商品を作成します。Store ID 必須

GET /products

商品一覧を取得します。

GET /products/{id}

特定の商品詳細を取得します。

PUT /products/{id}

商品情報を更新します。

注文（Orders）API

注文の取得、発送処理に関するエンドポイントです。

配送方法

種類 説明 手順
オフライン配送 許可された配送業者で発送 1. 商品を発送
2. offline fulfill エンドポイントで追跡番号を送信
オンライン配送 Joom Logistics等を利用 1. online fulfill エンドポイントで配送リクエスト
2. shipping label エンドポイントでラベル取得
3. 商品を発送
注意： オンライン配送は特定の発送元・配送先国、および特定の商品タイプでのみ利用可能です。一部のケースではオンライン配送が必須となります。

主要エンドポイント

GET /orders

注文一覧を取得します。

GET /orders/{id}

特定の注文詳細を取得します。

POST /orders/{id}/fulfill/offline

オフライン配送で注文を発送処理します。

POST /orders/{id}/fulfill/online

オンライン配送で注文を発送処理します。

POST /orders/{id}/modify-tracking

発送済み注文の追跡情報を変更します。
